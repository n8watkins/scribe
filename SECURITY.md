# Security Policy

## Supported versions

Security fixes are provided for the latest released version of Scribe.
Older releases are not supported, so users should update before reporting a problem that may already be fixed.

## Reporting a vulnerability

Please report suspected vulnerabilities through [GitHub private vulnerability reporting](https://github.com/n8watkins/scribe/security/advisories/new).
Include a clear description, affected versions, reproduction steps, impact, and any suggested mitigation you have identified.

Do not open a public issue, discussion, or pull request for an undisclosed vulnerability.
Keeping the report private gives maintainers time to investigate, prepare a fix, and coordinate disclosure.

For ordinary bugs that do not have security impact, use the repository's public issue tracker.
